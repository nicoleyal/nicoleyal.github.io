Step 2: Starting the Parametric Model

I started by measuring the circles in my existing construction kit so I could make a parametric model.



Then I used the circle component in Grasshopper to make the first part of the shape.



After that I made a rectangle and used number sliders to make the size adjustable. I measured the notch to figure out what size I should make my rectangle.
I also found an MD (multidimensional) slider which allowed me to move the rectangle around.




I used the polar array component to create a circular array of rectangles and used a number slider for the count to make it adjustable.




I made 3 more circles and used sliders to move them into the correct positions. I also started using the polar array functions to try to set them in the right places.




Step 3: Troubleshooting

I found the correct angles and relative x,y positions for the rectangles, but was left with a tiny adjustment that just wasn't possible with the way I had set things up.




Fortunately, Nadya posted the screencapture just in time, so I learned how to use the Move component. Thank you Nadya!




At this point I had everything ready to unite the shapes. Except everything I tried to unite or join the shapes wasn't working.




After a brief meltdown, I decided to bake everything to see if I could unite and trim everything in Rhino.




10 Google searches later, I finally found the right command - Curveboolean




So I moved the first shape aside and started working on a new one using the second thickness of material.
Things went more smoothly this time after learning a lot of lessons the first time around.
I measured my new material and adjusted the size of my rectangles to be a bit bigger to match the thicker material.




After following the same steps to bake and use the Curveboolean command, I made a linear array of both shapes to get ready to export it to Illustrator.
